GPIO Pins

The ESP32 DevKit v1 exposes 34 GPIO pins. Some have restrictions during boot — avoid them in your circuits.

GPIOSafe?Notes
GPIO 2✓On-board LED, ADC2_CH2
GPIO 4✓ADC2_CH0, touch T0
GPIO 5✓PWM, strapping pin (avoid at boot)
GPIO 6 — 11✗Connected to internal flash — never use
GPIO 12⚠Strapping pin — don't pull HIGH at boot
GPIO 15⚠Strapping pin — don't pull LOW at boot
GPIO 21✓I2C SDA (default)
GPIO 22✓I2C SCL (default)
GPIO 25✓DAC1, ADC2_CH8
GPIO 26✓DAC2, ADC2_CH9
GPIO 27✓ADC2_CH7
GPIO 32, 33✓ADC1_CH4 / CH5 (input-only on some boards)
GPIO 34, 35, 36, 39✓Input-only — no internal pull-up

PWM Pins

ESP32 has 16 PWM channels. Almost every GPIO supports PWM via the LEDC peripheral.

GPIOChannel suggestionCommon use
GPIO 2LEDC_CH0LED brightness
GPIO 4LEDC_CH1Buzzer / tone
GPIO 5LEDC_CH2Servo motor
GPIO 18LEDC_CH3RGB LED red
GPIO 19LEDC_CH4RGB LED green
GPIO 25LEDC_CH5RGB LED blue
GPIO 26LEDC_CH6Motor speed (via driver)
GPIO 27LEDC_CH7Fan speed
Quick tip Default PWM frequency is 5000 Hz with 8-bit resolution. Use ledcWrite(pin, duty) to set duty 0–255.

ADC Pins

I2C

Two-wire bus: SDA (data) and SCL (clock). Default pins work for most displays and sensors (OLED, MPU6050, BME280…).

SignalDefault pinNotes
SDAGPIO 21Use 4.7 kΩ pull-up to 3.3 V
SCLGPIO 22Use 4.7 kΩ pull-up to 3.3 V
Speed100 / 400 kHzStandard / Fast mode
DevicesUp to 127Each must have a unique 7-bit address
i2c_scan.ino
#include <Wire.h>
void setup() {
  Serial.begin(115200);
  Wire.begin(21, 22);
  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0) {
      Serial.printf("Found device at 0x%02X\n", addr);
    }
  }
}
void loop() {}

SPI

High-speed 4-wire bus: MOSI, MISO, SCK, CS. Default pins are safe for SD cards, RC522 RFID, and TFT displays.

SignalDefault pin (VSPI)Direction
MOSIGPIO 23Master → Slave
MISOGPIO 19Slave → Master
SCKGPIO 18Clock
CS / SSGPIO 5Chip select (active LOW)
HSPI alternative HSPI uses MOSI 13, MISO 12, SCK 14, CS 15. Use it when you need two SPI devices that conflict on VSPI pins.

UART

Asynchronous serial communication: TX and RX. ESP32 has three UARTs.

UARTTXRXCommon use
UART0GPIO 1GPIO 3Serial Monitor / USB
UART1GPIO 10GPIO 9GPS, external modules
UART2GPIO 17GPIO 16Bluetooth configuration
uart2.ino · second serial port
HardwareSerial MySerial(2);

void setup() {
  Serial.begin(115200);
  MySerial.begin(9600, SERIAL_8N1, 16, 17); // RX, TX
}

void loop() {
  if (MySerial.available()) {
    Serial.write(MySerial.read());
  }
}
